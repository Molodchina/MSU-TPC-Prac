1.1 Для проверки пароля нужно:
 - положительный просмотр вперед, который проверяет наличие хотя бы одной заглавной буквы,
 - положительный просмотр вперед, который проверяет наличие хотя бы одной строчной буквы,
 - положительный просмотр вперед, который проверяет наличие хотя бы одной цифры,
 - положительный просмотр вперед, который проверяет наличие хотя бы двух различных специальных символов,
 - отрицательный просмотр вперед, который проверяет отсутствие двух одинаковых символов подряд,
 - убедиться, что строка состоит только из латинских символов, цифр и специальных символов и имеет длину не менее 8 символов.
 
1.2 Для проверки корректности web цвета нужно написать реегулярное выражение для:
 - проверки корректности RGB цвета (комбинация либо из трёх целых чисел от 0 до 255, либо трёх процентных значений от 0% до 100%, перечисленных через запятую),
 - для проверки hex цвета (начинаются с решетки, затем либо 3, либо 6 подряд шестнадцатеричных цифр),
 - для проверки hsl (начинается с hsl, затем целое число от 0 до 360, потом s и l — процентные значения от 0% до 100%).
 
 1.3 Для токенизации математического выражения нужно строку на токены, определяя их тип:
 - сначала создадим регулярные выражения для каждого типа токенов,
 - объединить их в одно общее регулярное выражение, добавляя свой тип к каждому токену.

 1.4 Для проверки корректности даты нужно учитывать все указанные форматы дат, по максимуму объединяя форматы, чтобы уменьшить размеры регулярного выражения.
